# 角色管理功能 - 使用说明

## 功能概述

角色管理功能提供了可视化的界面来管理角色和分配菜单权限,让权限配置更加简单直观。

## 部署步骤

### 第一步:执行数据库脚本

```bash
# 添加角色管理菜单权限
mysql -u root -p kc < sql/add_role_menu.sql
```

### 第二步:清除Redis缓存

```bash
redis-cli
> DEL user:auth:admin
> EXIT
```

### 第三步:重启应用

重新启动应用后,admin用户登录即可在左侧菜单看到"角色管理"入口。

## 功能说明

### 1. 角色列表页面

**访问路径:** `/role/list`

**功能:**
- 查看所有启用的角色
- 新增角色
- 编辑角色信息
- 删除角色(系统默认角色不可删除)
- 为角色分配权限

**页面操作:**
- 点击"新增角色"按钮,在弹窗中填写角色信息
- 点击"分配权限"按钮,进入权限分配页面
- 点击"编辑"按钮,修改角色基本信息
- 点击"删除"按钮,删除角色(需要确认)

### 2. 权限分配页面

**访问路径:** `/role/permission/{roleId}`

**功能:**
- 以树形结构展示所有菜单
- 勾选/取消菜单权限
- 一键全选/取消全选
- 保存权限配置

**页面操作:**

#### 树形菜单结构
```
☑ 系统管理 (目录)
  ☑ 用户管理 (菜单)
  ☑ 接口凭据 (菜单)
  ☑ 角色管理 (菜单)
    ☑ 分配权限 (按钮)
    ☑ 新增角色 (按钮)
    ☑ 编辑角色 (按钮)
    ☑ 删除角色 (按钮)

☑ 业务工具 (目录)
  ☑ 模拟下单 (菜单)
  ☑ 坐标解析 (菜单)
```

#### 菜单类型说明
- **目录** (蓝色徽章) - 菜单分组,如"系统管理"、"业务工具"
- **菜单** (绿色徽章) - 可访问的页面,如"用户管理"
- **按钮** (浅蓝徽章) - 页面内的操作权限,如"新增"、"删除"

#### 操作步骤
1. 勾选需要分配的菜单(父节点勾选会自动包含子节点)
2. 点击"全选"或"取消全选"快速操作
3. 点击"保存权限"按钮保存配置
4. 保存成功后自动返回列表页面

## 使用示例

### 示例1:创建"运营人员"角色

**步骤:**
1. 登录admin账号
2. 进入"角色管理"页面
3. 点击"新增角色"
4. 填写信息:
   - 角色名称: 运营人员
   - 角色编码: ROLE_OPERATOR
   - 状态: 启用
   - 备注: 负责日常运营,只能访问业务工具
5. 点击"保存"
6. 在角色列表找到"运营人员",点击"分配权限"
7. 勾选:
   - ☑ 业务工具
     - ☑ 模拟下单
     - ☑ 坐标解析
8. 点击"保存权限"

**结果:** 创建了一个只能访问业务工具的运营人员角色

### 示例2:为现有用户分配新角色

**SQL操作:**
```sql
-- 查看用户当前角色
SELECT u.username, r.role_name 
FROM sys_user u
LEFT JOIN sys_user_role ur ON u.id = ur.user_id
LEFT JOIN sys_role r ON ur.role_id = r.id
WHERE u.username = 'test';

-- 为test用户分配运营人员角色(假设角色ID=3)
UPDATE sys_user_role 
SET role_id = 3 
WHERE user_id = (SELECT id FROM sys_user WHERE username = 'test');
```

**清除缓存:**
```bash
redis-cli
> DEL user:auth:test
```

**验证:** test用户重新登录后,权限会更新为运营人员角色的权限

## 添加新菜单权限

如果需要添加新的功能模块,需要同时添加菜单权限:

### SQL示例:添加"数据报表"菜单

```sql
-- 1. 添加目录
INSERT INTO sys_menu (parent_id, menu_name, menu_code, menu_type, url, icon, sort_order) VALUES
(0, '数据报表', 'menu:report', 1, '', 'bi-bar-chart-fill', 3);

-- 获取刚插入的ID(假设为12)
SET @report_menu_id = LAST_INSERT_ID();

-- 2. 添加子菜单
INSERT INTO sys_menu (parent_id, menu_name, menu_code, menu_type, url, icon, sort_order) VALUES
(@report_menu_id, '销售报表', 'menu:report:sales', 2, '/report/sales', 'bi-graph-up', 1),
(@report_menu_id, '用户报表', 'menu:report:user', 2, '/report/user', 'bi-people', 2);

-- 3. 为管理员角色分配权限
INSERT INTO sys_role_menu (role_id, menu_id) 
SELECT 1, id FROM sys_menu WHERE menu_code LIKE 'menu:report%';
```

## 注意事项

### 1. 缓存问题
修改角色权限后,**必须清除对应用户的Redis缓存**或等待30分钟自动过期:
```bash
redis-cli
> DEL user:auth:用户名
```

### 2. 权限生效时机
- 权限修改后,**已登录用户需要重新登录**才能生效
- 因为权限信息在登录时加载到Session中

### 3. 系统角色保护
- ID=1的系统管理员角色不可删除
- 建议在代码中保护其他核心角色

### 4. 权限编码规范
- 目录: `menu:模块名`
- 菜单: `menu:模块名:功能名`
- 按钮: `menu:模块名:功能名:操作`
- 角色: `ROLE_角色名`(大写)

## 常见问题

### Q1: 菜单不显示?
A: 检查:
1. 当前用户是否有该菜单的权限
2. Redis缓存是否已清除
3. 菜单的status字段是否为1(启用)

### Q2: 保存权限失败?
A: 检查浏览器控制台错误信息,可能是:
1. 没有menu:role:permission权限
2. 数据库连接问题
3. CSRF Token问题

### Q3: 如何批量为用户分配角色?
A: 使用SQL批量操作:
```sql
-- 为所有status=1的用户分配普通用户角色
INSERT INTO sys_user_role (user_id, role_id)
SELECT id, 2 FROM sys_user 
WHERE status = 1 
AND id NOT IN (SELECT user_id FROM sys_user_role);
```

### Q4: 权限树能否实现级联选择?
A: 当前版本需要手动勾选每个节点。可以后续添加JavaScript实现:
- 勾选父节点自动勾选所有子节点
- 取消父节点自动取消所有子节点
- 子节点全选自动勾选父节点
