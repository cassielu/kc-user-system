# 操作日志功能说明

## 📋 功能概述

操作日志功能用于记录系统中所有重要操作，包括：
- 用户角色分配
- 角色权限分配
- 操作成功/失败状态
- 操作耗时
- 操作用户IP地址

## 💡 功能特性

### 1. 异步记录
- 使用 `@Async` 异步记录日志
- 不影响主业务流程性能
- 日志记录失败不影响主操作

### 2. 自动采集
- 自动获取当前登录用户
- 自动获取客户端IP地址
- 自动获取浏览器信息
- 自动计算操作耗时

### 3. 智能截断
- 请求参数最大2000字符
- 错误信息最大2000字符
- 防止数据库字段溢出

### 4. 查询过滤
- 按操作用户查询
- 按操作描述查询
- 分页显示（每页20条）

## 🔧 扩展日志记录

在任何Controller方法中添加日志记录：

```java
@Autowired
private OperationLogHelper operationLogHelper;

@PostMapping("/some-operation")
public Map<String, Object> someOperation() {
    long startTime = System.currentTimeMillis();
    try {
        // 业务逻辑
        
        // 记录成功日志
        long costTime = System.currentTimeMillis() - startTime;
        operationLogHelper.logSuccess(
            "操作描述",
            "POST /some-operation",
            "param1=value1",
            costTime
        );
    } catch (Exception e) {
        // 记录错误日志
        long costTime = System.currentTimeMillis() - startTime;
        operationLogHelper.logError(
            "操作失败",
            "POST /some-operation",
            "param1=value1",
            e.getMessage(),
            costTime
        );
    }
}
```

## ⚠️ 注意事项

1. **性能考虑**
   - 日志记录是异步的，不影响主流程
   - 建议定期清理历史日志（如保留90天）

2. **权限控制**
   - 需要 `menu:log:list` 权限才能查看
   - 默认只有超级管理员有此权限

3. **数据安全**
   - 敏感参数会被记录（如密码）
   - 建议在生产环境过滤敏感信息

4. **存储空间**
   - TEXT字段可能占用较多空间
   - 建议定期归档或删除旧日志
