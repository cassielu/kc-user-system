# 用户角色分配功能 - 使用说明

## 功能概述

在用户管理页面中增加了可视化的角色分配功能,管理员可以直接在界面上为用户分配角色,无需编写SQL。

---

## 使用步骤

### 1. 进入用户管理页面

使用 **admin** 账号登录系统,点击左侧菜单:
**系统管理** → **用户管理**

### 2. 查看用户角色

在用户列表表格中,新增了**"角色"**列,显示每个用户已分配的角色:
- 已分配角色:显示角色名称的蓝色徽章
- 未分配角色:显示"未分配"
- 无可用角色:显示"无可用角色"

### 3. 为用户分配角色

#### 步骤1:点击"角色"按钮
在用户列表的操作列,找到要分配角色的用户,点击 **"角色"** 按钮。

#### 步骤2:选择角色
在弹出的对话框中:
- 顶部显示用户信息
- 中间显示所有可用的角色列表(复选框)
- 已分配的角色会自动勾选
- 可以勾选/取消多个角色

#### 步骤3:保存
点击 **"保存"** 按钮,系统会:
1. 保存角色分配
2. 自动清除用户认证缓存
3. 刷新角色显示
4. 提示"角色分配成功!用户需重新登录生效。"

### 4. 验证权限

用户需要**重新登录**后,新的角色权限才会生效。

## 技术实现

### 后端接口

#### 1. 获取用户角色信息
```
GET /user/role-info/{userId}
```
**返回:**
```json
{
  "allRoles": [
    {"id": 1, "roleName": "系统管理员", "roleCode": "ROLE_ADMIN", "remark": "拥有所有权限"},
    {"id": 2, "roleName": "普通用户", "roleCode": "ROLE_USER", "remark": "只能访问业务工具"}
  ],
  "userRoleIds": [1]
}
```

#### 2. 分配用户角色
```
POST /user/assign-role/{userId}
```
**参数:**
- `roleIds`: 角色ID列表(可多个)

**返回:**
```json
{
  "success": true,
  "message": "角色分配成功"
}
```

### 前端实现

1. **页面加载** - 自动获取并显示所有用户的角色
2. **角色分配弹窗** - Bootstrap Modal + 复选框列表
3. **AJAX请求** - Fetch API异步操作
4. **实时更新** - 保存成功后自动刷新角色显示

## 使用示例

### 示例1:为test用户分配管理员角色

1. 登录admin账号
2. 进入用户管理页面
3. 找到test用户,点击"角色"按钮
4. 在弹窗中勾选"系统管理员"
5. 点击"保存"
6. test用户重新登录后,拥有管理员权限

### 示例2:为新用户分配普通用户角色

1. 创建新用户(用户管理 → 新增用户)
2. 在新用户的操作列点击"角色"按钮
3. 勾选"普通用户"
4. 点击"保存"
5. 新用户登录后,拥有普通用户权限

### 示例3:为用户分配多个角色

系统支持一个用户拥有多个角色,权限会合并:

1. 点击用户的"角色"按钮
2. 同时勾选"普通用户"和"运营人员"
3. 点击"保存"
4. 用户将拥有两个角色的所有权限

## 权限模型

```
用户(sys_user) 
  ↓ 多对多
角色(sys_role)
  ↓ 一对多  
菜单权限(sys_menu)
```

**核心逻辑:**
- 一个用户可以有多个角色
- 一个角色可以分配给多个用户
- 用户的权限 = 所有角色权限的并集

## 注意事项

### 1. 缓存机制
- 角色分配后会自动清除该用户的认证缓存
- 但**已登录的用户需要重新登录**才能生效
- 因为权限信息在登录时加载到Session中

### 2. 角色保护
- 可以为用户分配多个角色
- 也可以不分配任何角色(但会导致无权限)
- 建议至少分配一个角色

### 3. 权限变更
- 修改角色的权限(角色管理页面)会影响所有拥有该角色的用户
- 修改用户的角色(用户管理页面)只影响该用户

### 4. 安全考虑
- 只有admin用户可以操作角色分配
- 使用了Spring Security的权限控制
- 不允许用户修改自己的角色

## 数据库操作(备用)

如果界面操作遇到问题,也可以通过SQL直接操作:

```sql
-- 查看用户的角色
SELECT u.username, r.role_name, r.role_code
FROM sys_user u
LEFT JOIN sys_user_role ur ON u.id = ur.user_id
LEFT JOIN sys_role r ON ur.role_id = r.id
WHERE u.username = 'test';

-- 为用户分配角色
INSERT INTO sys_user_role (user_id, role_id) VALUES (用户ID, 角色ID);

-- 删除用户的某个角色
DELETE FROM sys_user_role WHERE user_id = 用户ID AND role_id = 角色ID;

-- 清空用户的所有角色
DELETE FROM sys_user_role WHERE user_id = 用户ID;
```

清除缓存:
```bash
redis-cli
> DEL user:auth:用户名
```

## 常见问题

### Q1: 分配角色后权限没变化?
A: 用户需要**重新登录**,因为权限信息在登录时加载。

### Q2: 可以为用户分配多个角色吗?
A: 可以,系统支持多角色,权限会合并。

### Q3: 如何查看某个角色的权限?
A: 进入"角色管理"页面,点击该角色的"分配权限"按钮查看。

### Q4: 删除用户后角色会自动删除吗?
A: 会的,MyBatis Plus的级联删除会自动清理sys_user_role表的关联数据。
