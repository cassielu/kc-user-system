# 菜单权限管理系统 - 使用说明

## 功能概述

本系统实现了基于 RBAC(基于角色的访问控制) 的菜单权限管理,支持:
- 用户分配角色
- 角色关联菜单权限
- 根据用户权限动态显示菜单
- 后端接口权限保护

## 数据库初始化

### 方式一:全新安装
执行完整初始化脚本:
```bash
mysql -u root -p < sql/init.sql
```

### 方式二:已有系统升级
执行升级脚本(不会破坏现有数据):
```bash
mysql -u root -p kc < sql/upgrade_permission.sql
```

## 权限模型

### 数据结构
```
用户(sys_user) → 用户角色关联(sys_user_role) → 角色(sys_role)
角色(sys_role) → 角色菜单关联(sys_role_menu) → 菜单(sys_menu)
```

### 默认角色

1. **ROLE_ADMIN(系统管理员)**
   - 拥有所有菜单权限
   - 可访问:系统管理、业务工具下的所有功能

2. **ROLE_USER(普通用户)**
   - 只能访问业务工具
   - 可访问:模拟下单、坐标解析

## 使用方式

### 1. 用户登录验证

使用不同账号登录测试:
- **admin/123456** - 应看到所有菜单
- **test/123456** - 应只看到"业务工具"菜单

### 2. 为用户分配角色

直接在数据库中操作用户角色关联:

```sql
-- 查看用户已分配的角色
SELECT u.username, r.role_name, r.role_code
FROM sys_user u
LEFT JOIN sys_user_role ur ON u.id = ur.user_id
LEFT JOIN sys_role r ON ur.role_id = r.id
WHERE u.username = 'test';

-- 为test用户分配管理员角色
UPDATE sys_user_role SET role_id = 1 
WHERE user_id = (SELECT id FROM sys_user WHERE username = 'test');

-- 为test用户分配普通用户角色
UPDATE sys_user_role SET role_id = 2 
WHERE user_id = (SELECT id FROM sys_user WHERE username = 'test');
```

### 3. 清除缓存

由于权限信息使用了 Redis 缓存,修改权限后需要清除缓存才能生效:

```sql
-- 方式1:通过Redis客户端清除
redis-cli
> DEL user:auth:admin
> DEL user:auth:test

-- 方式2:重新登录(缓存TTL=30分钟,等待自动过期)
```

### 4. 自定义菜单权限

#### 添加新菜单

```sql
-- 在"系统管理"下添加新菜单
INSERT INTO sys_menu (parent_id, menu_name, menu_code, menu_type, url, icon, sort_order) VALUES
(1, '角色管理', 'menu:role:list', 2, '/role/list', 'bi-shield-fill', 3);

-- 为管理员角色分配新菜单权限
INSERT INTO sys_role_menu (role_id, menu_id) 
VALUES (1, (SELECT id FROM sys_menu WHERE menu_code = 'menu:role:list'));
```

#### 添加新角色

```sql
-- 创建"运营人员"角色
INSERT INTO sys_role (role_name, role_code, status, remark) VALUES
('运营人员', 'ROLE_OPERATOR', 1, '只能访问业务工具');

-- 为该角色分配菜单权限
INSERT INTO sys_role_menu (role_id, menu_id)
SELECT (SELECT id FROM sys_role WHERE role_code = 'ROLE_OPERATOR'), id
FROM sys_menu WHERE menu_code IN ('menu:business', 'menu:order:simulate');

-- 为用户分配该角色
INSERT INTO sys_user_role (user_id, role_id)
VALUES ((SELECT id FROM sys_user WHERE username = '新用户名'), 
        (SELECT id FROM sys_role WHERE role_code = 'ROLE_OPERATOR'));
```

## 前端权限控制

### Thymeleaf 模板中使用

布局页面 [layout.html](file:///d:/workspace-kc/kc-user-system/src/main/resources/templates/layout.html) 已使用 `sec:authorize` 标签:

```html
<!-- 根据权限显示/隐藏菜单 -->
<li class="nav-item" sec:authorize="hasAuthority('menu:user:list')">
    <a class="nav-link" th:href="@{/user/list}">
        <i class="bi bi-people-fill"></i>用户管理
    </a>
</li>
```

### 其他页面中使用

在任何 Thymeleaf 模板中都可以使用:

```html
<!-- 显示/隐藏按钮 -->
<button sec:authorize="hasAuthority('menu:user:add')" class="btn btn-primary">
    新增用户
</button>

<!-- 显示/隐藏整个区域 -->
<div sec:authorize="hasAuthority('menu:system')">
    系统管理相关功能
</div>
```

## 后端权限保护

### Controller 方法级别

在需要权限保护的方法上添加 `@PreAuthorize` 注解:

```java
@PreAuthorize("hasAuthority('menu:user:list')")
@GetMapping("/user/list")
public String listPage() {
    // 只有拥有 menu:user:list 权限的用户才能访问
}

@PreAuthorize("hasAuthority('menu:user:add')")
@PostMapping("/user/add")
public String addUser(@Valid SysUser user) {
    // 只有拥有 menu:user:add 权限的用户才能新增
}
```

### 权限标识规范

建议采用以下命名规范:
- `menu:模块:操作` - 菜单权限,如 `menu:user:list`
- `ROLE_角色名` - 角色权限,如 `ROLE_ADMIN`

## 技术实现要点

1. **Spring Security 集成**
   - 用户登录时动态加载角色和菜单权限
   - 权限信息注入到 `UserDetails` 的 `GrantedAuthority` 列表

2. **Thymeleaf 安全标签**
   - 使用 `thymeleaf-extras-springsecurity5` 依赖
   - `sec:authorize="hasAuthority('权限编码')"` 控制元素显示

3. **缓存策略**
   - 权限信息缓存在 Redis 中,TTL=30分钟
   - 修改权限后需清除 `user:auth:{username}` 缓存

4. **方法级安全**
   - 通过 `@EnableGlobalMethodSecurity(prePostEnabled = true)` 启用
   - 使用 `@PreAuthorize` 注解保护接口

## 常见问题

### Q1: 修改权限后菜单没有变化?
A: 需要清除 Redis 缓存或重新登录。权限信息有30分钟缓存。

### Q2: 如何查看所有菜单权限?
```sql
SELECT menu_code, menu_name FROM sys_menu WHERE status = 1 ORDER BY sort_order;
```

### Q3: 如何查看某用户的所有权限?
```sql
SELECT DISTINCT m.menu_code, m.menu_name
FROM sys_menu m
INNER JOIN sys_role_menu rm ON m.id = rm.menu_id
INNER JOIN sys_user_role ur ON rm.role_id = ur.role_id
WHERE ur.user_id = (SELECT id FROM sys_user WHERE username = 'admin');
```

### Q4: 没有权限访问时会怎样?
A: Spring Security 会返回 HTTP 403 Forbidden 错误。
