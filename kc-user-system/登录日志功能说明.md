# 登录日志功能说明

## 📋 功能概述

登录日志功能用于记录所有用户的登录行为，包括：
- 登录成功/失败
- 登录时间
- 登录IP地址
- 浏览器信息
- 失败原因

## 💡 功能特性

### 1. 自动记录
- 登录成功自动记录
- 登录失败自动记录
- 无需手动调用

### 2. 异步处理
- 使用 `@Async` 异步记录
- 不影响登录流程性能
- 日志记录失败不影响登录

### 3. 智能识别
- 自动获取客户端IP
- 自动获取浏览器信息
- 智能识别失败原因

### 4. 失败分类
| 异常类型 | 提示信息 |
|---------|---------|
| DisabledException | 账号已停用 |
| UsernameNotFoundException | 用户不存在 |
| BadCredentialsException | 密码错误 |
| 其他 | 登录失败: 详细原因 |

### 5. 查询过滤
- 按用户名查询
- 按登录状态查询（成功/失败）
- 分页显示（每页20条）

## 🔧 技术实现

### 登录流程

```
用户提交登录表单
    ↓
Spring Security 认证
    ↓
成功 → CustomLoginSuccessHandler → 记录成功日志 → 跳转首页
失败 → CustomLoginFailureHandler → 记录失败日志 → 跳转登录页
```

### SecurityConfig配置

```java
.formLogin()
    .loginPage("/login")
    .loginProcessingUrl("/login")
    .successHandler(customLoginSuccessHandler)  // 自定义成功处理器
    .failureHandler(customLoginFailureHandler)  // 自定义失败处理器
    .permitAll()
```

## 🔍 使用场景

### 1. 安全审计
- 查看是否有异常登录
- 监控登录失败次数
- 发现暴力破解行为

### 2. 问题排查
- 用户反馈无法登录
- 查看具体失败原因
- 定位问题根源

### 3. 数据分析
- 统计登录频率
- 分析登录时间分布
- 了解用户活跃度

---

## ⚠️ 注意事项

1. **性能考虑**
   - 日志记录是异步的，不影响登录性能
   - 建议定期清理历史日志（如保留90天）

2. **权限控制**
   - 需要 `menu:log:login` 权限才能查看
   - 默认只有超级管理员有此权限

3. **隐私保护**
   - 记录IP地址和浏览器信息
   - 符合网络安全审计要求

4. **存储空间**
   - 登录日志可能增长较快
   - 建议定期归档或删除旧日志

## 📝 SQL查询示例

### 查看今天的登录记录
```sql
SELECT * FROM sys_login_log 
WHERE DATE(login_time) = CURDATE() 
ORDER BY login_time DESC;
```

### 查看登录失败次数最多的用户
```sql
SELECT username, COUNT(*) as fail_count 
FROM sys_login_log 
WHERE status = 0 
GROUP BY username 
ORDER BY fail_count DESC 
LIMIT 10;
```

### 查看某个IP的登录记录
```sql
SELECT * FROM sys_login_log 
WHERE ip = '192.168.1.100' 
ORDER BY login_time DESC;
```

### 统计每天的登录情况
```sql
SELECT 
    DATE(login_time) as login_date,
    SUM(CASE WHEN status = 1 THEN 1 ELSE 0 END) as success_count,
    SUM(CASE WHEN status = 0 THEN 1 ELSE 0 END) as fail_count
FROM sys_login_log 
GROUP BY DATE(login_time)
ORDER BY login_date DESC;
```
